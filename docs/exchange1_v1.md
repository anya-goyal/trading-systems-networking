# Exchange 1 API Documentation
**Version:** 1.0  
**Protocols:** UDP (Market Data), TCP (Order Entry & Snapshots)

---

## 1. General Conventions
* **Encoding:** All messages are transmitted as `UTF-8` encoded strings.
* **Delimiters:** * Field delimiter: Pipe character `|`
  * Message delimiter (TCP only): Newline character `\n`
* **Pricing:**
  * *PITCH (UDP):* Prices are fixed-point integers (Price * 10000). E.g., `$150.25` is `1502500`.
  * *Mini-FIX (TCP):* Prices are standard floating-point decimals. E.g., `150.25`.
* **Timestamps:** Unix epoch milliseconds.

---

## 2. Market Data Feed (Simplified PITCH)
**Transport:** UDP Multicast/Broadcast  
**Direction:** Exchange -> Subscribers  
**Description:** A low-latency, unacknowledged firehose of public order book events. 

### Message Structure
All UDP messages follow a strict position-based format separated by `|`:
`Sequence_Number | Timestamp | Message_Type | [Payload]`

### Supported Message Types

#### System Event (`S`)
Indicates a change in the exchange's trading state.
* **Format:** `Seq | Time | S | EventCode`
* **Event Codes:** `O` (Open), `C` (Close), `H` (Halt)
* **Example:** `1|1711805400000|S|O`

#### Add Order (`A`)
Broadcasts a newly resting limit order on the visible book.
* **Format:** `Seq | Time | A | OrderID | Symbol | AssetType | Side | Shares | PriceInt`
* **Fields:**
  * `OrderID`: 64-bit integer generated by the exchange.
  * `AssetType`: String or character indicating the asset class (e.g., `EQ` for Equity, `OPT` for Options, `FUT` for Futures).
  * `Side`: `B` (Buy) or `S` (Sell).
* **Example:** `2|1711805400050|A|101|AAPL|EQ|B|100|1502500`


#### Order Executed (`E`)
Indicates a resting order was matched. (Note: Does not include Symbol; subscribers must maintain a local `OrderID` mapping).
* **Format:** `Seq | Time | E | OrderID | ExecutedShares`
* **Example:** `3|1711805400100|E|101|50`

#### Order Canceled (`X`)
Indicates a resting order was removed or reduced in size.
* **Format:** `Seq | Time | X | OrderID | CanceledShares`
* **Example:** `4|1711805400150|X|101|50`

## 3. Order Entry Gateway (Mini-FIX)
**Transport:** TCP  
**Direction:** Bidirectional (Strategy <-> Exchange)  
**Description:** A reliable, stateful connection for submitting private orders and receiving execution reports.

### Message Structure
Messages are standard `Tag=Value` pairs separated by `|` and terminated by `\n`.
* **Required Header:** Every message must begin with `35=[MsgType]`.

### Standard Tag Dictionary
| Tag | Name | Type | Description |
| :--- | :--- | :--- | :--- |
| **`35`** | `MsgType` | Char | Determines the message purpose (`D`, `F`, `8`, `V`). |
| **`11`** | `ClOrdID` | String | Unique ID generated by the client strategy. |
| **`37`** | `OrderID` | Int | Unique ID generated by the exchange. |
| **`41`** | `OrigClOrdID`| String | Used in cancels to reference the target order. |
| **`55`** | `Symbol` | String | Ticker symbol (e.g., `AAPL`). |
| **`54`** | `Side` | Int | `1` = Buy, `2` = Sell. |
| **`38`** | `OrderQty` | Int | Total shares requested. |
| **`44`** | `Price` | Float | Limit price (e.g., `150.25`). |
| **`59`** | `TimeInForce`| Int | `0` = GTC (Day), `3` = IOC (Immediate or Cancel). |
| **`150`**| `ExecType` | Char | `0`=New, `1`=Partial, `2`=Fill, `4`=Cancel, `8`=Reject. |
| **`39`** | `OrdStatus` | Char | Current state (matches ExecType codes above). |
| **`14`** | `CumQty` | Int | Total shares executed across the order's lifetime. |
| **`31`** | `LastPx` | Float | Price of the most recent execution. |
| **`32`** | `LastQty` | Int | Shares executed in the most recent transaction. |

### Client-to-Exchange Endpoints

#### New Order Single (`35=D`)
Submits a new limit order to the matching engine.
* **Format:** `35=D|11=[ClOrdID]|55=[Symbol]|54=[Side]|38=[Qty]|44=[Price]|59=[TIF]\n`
* **Example:** `35=D|11=STRAT1-01|55=AAPL|54=1|38=100|44=150.25|59=0\n`

#### Order Cancel Request (`35=F`)
Attempts to pull an active resting order from the book.
* **Format:** `35=F|11=[NewCancelID]|41=[OrigClOrdID]|55=[Symbol]\n`
* **Example:** `35=F|11=CANC-01|41=STRAT1-01|55=AAPL\n`

### Exchange-to-Client Endpoints

#### Execution Report (`35=8`)
Sent by the exchange to acknowledge orders, report fills, or confirm cancellations.
* **Format:** `35=8|11=[ClOrdID]|37=[ExchID]|150=[ExecType]|39=[Status]|[Fill Data...]\n`
* **Example (Acknowledgment):** `35=8|11=STRAT1-01|37=999|150=0|39=0\n`
* **Example (Full Fill):** `35=8|11=STRAT1-01|37=999|150=2|39=2|14=100|31=150.25|32=100\n`
* **Example (Reject/IOC Miss):** `35=8|11=STRAT1-02|37=1000|150=4|39=4\n`
